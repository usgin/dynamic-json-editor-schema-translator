Dynamic Metadata JSON editor and Schema Translator (DyJest)

The Dygest editor provides a visual tree diagram for metadata in a JSON format. The color coding provides information about whether the data meets schema requirements (red branch segments), and options that can be performed. This schema(s) for this editor is a json file that can be modified to meet varying metdata editing requirements and sources.

Using the Editor

Header Functions
  • Enter your name - To be saving edits, enter your name that will be saved as part of the metadata record. Click on the Editor name, edit, then click to save.
  • json - The JSON button will display your edits in the "diff' format required by the pipeline processing.
  • Help - You are here.

DDH Metadata Site Map

The elements of a metadata record are viewed and edited for the Data Discovery Hub are edited in seven pages

Pages are accessed by selecting a Blue Node, or from the drop down list in the right navigation panel.

How to Navigate the Metadata Tree

Mouse
  • Zoom In & Zoom Out - use the Mouse Scroll wheel to zoom in and out, or double-click (in) and shift double-click(out)
  • Pan - Click and hold background to drag the tree to desired location
  • End Point Nodes- Click to edit
  • Parent Nodes - Click to collapse or expand, right click for the edit menu.

Metadata Tree Display Legend

Links and Nodes
  • End Point Node - is the last element in a tree, which can be edited. If the node has a value, the value will be displayed, else it will show the name of the field
  • Parent Nodes - Objects and Arrays - Objects are Blue circles with children. Arrays are Red circles with children. If the schema allows creating additional elements in the object, right click to for the menu to create, copy or delete elements
  • Links are the lines that connect the nodes, showing the parent child relationship defined in the schema.
  • Links for Required Data - Nodes that are required data elements will be either Green (populated) or Red (missing data).
  • Links for Optional Data - Links that are defined as optional data will either be Gold (populated) or Grey (empty).

Editing Metadata

Data Types that can be edited:
  • Text - Edit, delete or add text
  • Large Text Areas - resizable object to handle large text area.
  • Arrays - Right click on an array (the Red Circles ) and you can add new elements or delete elements in the array
  • Parent Nodes (Objects) - right click on an array (the Blue Circles ) and you can add a new element or delete element elements to the object. Objects are strongly type, you can only add objects if they do not already exist. To allow multiple objects of the same type, use the array element.
  • Select lists - Select list are populated by a url reference list, can be external to system, such as data type registry, dictionary, or thesaurus.
  • Typehead Lists - Similar to select list in functionality, provides exernally referenced Typeahead lookup.
  • Map Bounding Box - graphic map display that allow you to view and modify a bounding box.
  • Dates - View and pick a date from calendar
  • GUIDS - Guids can only be created and not edited. Click on an empty guid to create a new one

Metadata Schemas

  • JSON schemas translate the incoming data to the tree structure displayed, and transform it back when saved. To change the tree view and the data to be edited, requires a new json schema to be uploaded.
  • JSON Data Dictionaries - populate pick lists for select fields.
